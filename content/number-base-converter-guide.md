## 什么是进制转换？

**进制转换（Base Conversion）** 是将一个数字从一种进位制表示转换为另一种进位制表示的过程。计算机科学中，最常见的进制包括二进制（Binary，Base-2）、八进制（Octal，Base-8）、十进制（Decimal，Base-10）和十六进制（Hexadecimal，Base-16）。理解不同进制之间的转换方法是程序员、网络工程师和计算机学生的必备技能。

### 常见进制速览

| 进制 | 基数 | 使用的数字 | 常见用途 |
|------|:----:|-----------|---------|
| **二进制 (BIN)** | 2 | 0-1 | 计算机底层、位运算、数字电路 |
| **八进制 (OCT)** | 8 | 0-7 | Unix 文件权限、早期计算机系统 |
| **十进制 (DEC)** | 10 | 0-9 | 人类日常生活、数学计算 |
| **十六进制 (HEX)** | 16 | 0-9, A-F | 内存地址、颜色值、MAC地址、编码 |

## 进制转换基础方法

### 1. 其他进制 → 十进制

将每一位的数字乘以该位对应的权重（基数^位置），然后求和。

**二进制转十进制示例：** `1011₂ = 1×2³ + 0×2² + 1×2¹ + 1×2⁰ = 8 + 0 + 2 + 1 = 11₁₀`

**十六进制转十进制示例：** `FF₁₆ = 15×16¹ + 15×16⁰ = 240 + 15 = 255₁₀`

### 2. 十进制 → 其他进制

使用除法取余法：将十进制数不断除以目标基数，将余数从下往上排列。

**十进制转二进制示例：** 25₁₀ → 11001₂
- 25 ÷ 2 = 12 余 1
- 12 ÷ 2 = 6 余 0
- 6 ÷ 2 = 3 余 0
- 3 ÷ 2 = 1 余 1
- 1 ÷ 2 = 0 余 1
- 从下往上读取：11001

### 3. 二进制 ↔ 十六进制（快速转换）

每 4 位二进制数对应 1 位十六进制数：
- `0000₂ = 0₁₆`、`0001₂ = 1₁₆`、...、`1010₂ = A₁₆`、...、`1111₂ = F₁₆`

**示例：** `10111100₂ → 1011₁₆ = B₁₆, 1100₁₆ = C₁₆ → BC₁₆`

### 4. 二进制 ↔ 八进制（快速转换）

每 3 位二进制数对应 1 位八进制数：
- `000₂ = 0₈`、`001₂ = 1₈`、...、`111₂ = 7₈`

**示例：** `101110₂ → 101₈ = 5₈, 110₈ = 6₈ → 56₈`

## 程序员常遇到的进制转换场景

### 1. CSS 颜色值转换

网页开发中颜色用十六进制表示（如 `#FF8800`），但有时需要知道 RGB 分量值：
- `FF₁₆ = 255`（红色分量）
- `88₁₆ = 136`（绿色分量）
- `00₁₆ = 0`（蓝色分量）
- 所以 `#FF8800 = rgb(255, 136, 0)`

### 2. 内存地址和指针

调试时看到的内存地址如 `0x7ffeefbff5a8` 是十六进制表示，了解其十进制值有助于理解内存布局。

### 3. IP 地址与子网掩码

IP 地址 `192.168.1.1` 的二进制形式是：
`11000000.10101000.00000001.00000001`
子网掩码计算需要理解二进制位运算。

### 4. 位掩码和权限控制

Unix/Linux 文件权限用八进制表示（如 `chmod 755`）：
- `7₈ = 111₂`（读+写+执行）
- `5₈ = 101₂`（读+执行）
- `5₈ = 101₂`（读+执行）

### 5. Unicode 和字符编码

查看字符的 Unicode 码点时常遇到十六进制值，如 `U+4E2D`（"中"字）对应的十进制是 `20013`。

### 快速心算

- **2ⁿ 的值**：2¹⁰ = 1024（1K），2²⁰ = 1048576（1M）
- **十六进制转二进制**：一位十六进制 = 四位二进制，`A=1010`、`B=1011`、`C=1100`、`D=1101`、`E=1110`、`F=1111`
- **八进制转二进制**：一位八进制 = 三位二进制
