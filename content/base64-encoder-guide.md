## 什么是 Base64？

Base64 是一种用 64 个可打印字符来表示二进制数据的编码方案。它使用字母 A-Z、a-z、数字 0-9 以及符号 `+` 和 `/` 共 64 个字符（加上填充字符 `=`），将任意二进制数据转换为纯文本格式。

Base64 的核心理念是：**每 3 个字节 (24 bits) 的数据被分成 4 组，每组 6 bits，对应一个可打印字符**。

### Base64 的历史

Base64 最初在 RFC 1421 中定义（1993 年），后来在 RFC 2045（MIME）中被广泛采用。如今它是互联网基础设施中不可或缺的一部分。

## 为什么需要 Base64 编码？

### 1. 在文本协议中传输二进制数据

HTTP、JSON、XML、HTML 都是文本协议。如果你想在 JSON 响应中返回一张图片或一个二进制文件，直接用二进制数据会破坏协议格式。Base64 将二进制数据转换为安全的 ASCII 文本，解决了这个问题。

### 2. Data URL 嵌入资源

在 HTML 和 CSS 中，可以使用 `data:` URL 直接嵌入资源。例如：

```html
<img src="data:image/png;base64,iVBORw0KGgo...">
```

这样做的好处是减少 HTTP 请求数，适合小尺寸资源（如图标、字体）。

### 3. JWT Token

JSON Web Token (JWT) 使用 Base64 URL 安全编码来编码令牌的 Header 和 Payload 部分。

### 4. 邮件附件 (MIME)

电子邮件协议主要处理文本。Base64 让邮件附件（图片、文件）可以通过纯文本邮件传输。

## Base64 编码的特点

| 特性 | 说明 |
|------|------|
| **数据膨胀** | Base64 编码后数据大小增加约 **33%**（每 3 字节 → 4 字符） |
| **可逆** | 编码后的 Base64 字符串可以解码还原为原始数据 |
| **不安全** | Base64 **不是加密**，编码后的数据可以被任何人解码 |
| **无密钥** | 不需要密钥，纯粹是格式转换 |

## 常见问题 (FAQ)

### Q: Base64 是加密吗？
**不是**。Base64 只是一种编码方式，类似于把十进制数字转成十六进制。任何人都能解码 Base64，不要用它保护敏感信息。

### Q: 为什么 Base64 以 `=` 结尾？
Base64 要求输入数据长度是 3 的倍数。如果不是，会用 `=` 填充。一个 `=` 表示缺 1 字节，两个 `=` 表示缺 2 字节。

### Q: 使用 Base64 文件编码后，文件会变大吗？
是的。Base64 编码后数据体积约增大 33%。对于大文件（>1MB），建议使用二进制传输而不是 Base64。

### Q: URL 安全的 Base64 是什么？
标准 Base64 包含 `+` 和 `/` 字符，在 URL 中有特殊含义。URL 安全 Base64 将 `+` 替换为 `-`，将 `/` 替换为 `_`，并去掉 `=` 填充。
