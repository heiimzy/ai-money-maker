## 什么是 JSON 格式化？

JSON（JavaScript Object Notation）是目前 Web 开发中最主流的数据交换格式。无论是 REST API 的请求响应、前端配置文件（如 `package.json`、`tsconfig.json`），还是数据存储场景，JSON 无处不在。然而，在实际开发中，我们经常会遇到**压缩成一行**的 JSON 数据，几乎无法人工阅读和调试。这时，**JSON 格式化工具**就显得尤为重要。

## 为什么需要在线 JSON 格式化工具？

### 1. 调试 API 响应
在前后端分离的开发模式下，后端返回的 JSON 数据通常经过压缩以节省带宽。使用在线 JSON 格式化工具，可以一键将压缩的 JSON **美化**成带层级缩进的结构，让数据一目了然。

### 2. 验证 JSON 语法
手写 JSON 配置时容易出现多余的逗号、缺少引号、花括号不匹配等语法错误。好的 JSON 校验工具不仅告诉你"有错误"，还能**精确定位到错误发生的行和列**，大幅提升排错效率。

### 3. 数据压缩与传输优化
在线 JSON 压缩功能可以去除所有不必要的空白字符，将 JSON 体积缩小 30%-50%，适合用于生产环境的数据传输。

## JSON 使用技巧

### 正确使用 JSON 的 5 个要点

1. **始终使用双引号** — JSON 要求 key 和 string 值必须用双引号，单引号是非法的
2. **结尾无逗号** — 最后一个元素后面不能有逗号，这是最常见的错误之一
3. **不支持注释** — JSON 本身不支持 `//` 或 `/* */` 注释，如需注释可以使用 JSONC 格式
4. **使用在线工具验证** — 在将 JSON 配置部署到生产环境前，先用工具验证
5. **嵌套不要过深** — 超过 4 层嵌套的 JSON 可读性急剧下降，应考虑重构数据结构
