# 时间戳转换完整指南：Unix Timestamp 与日期时间互转详解

## 什么是 Unix 时间戳？

**Unix 时间戳（Unix Timestamp）** 是计算机系统中通用的时间表示方式，定义为从 **1970年1月1日 00:00:00 UTC**（协调世界时，也称 Unix Epoch）开始经过的秒数（或毫秒数）。无论你身处哪个时区，Unix 时间戳在全球范围内都是统一的。

简单来说，时间戳就是一个不断递增的数字，代表从 Unix 纪元到当前时刻的时间差：

- **秒级时间戳**：10位数字，如 `1715712000`
- **毫秒级时间戳**：13位数字，如 `1715712000000`
- **微秒/纳秒级**：16-19位数字，常见于高性能计算场景

### 常见使用场景

#### 1. 数据库时间字段调试
许多数据库（MySQL、PostgreSQL、MongoDB）使用 Unix 时间戳存储时间。排查问题时，需要将时间戳转换为人类可读的日期格式。


#### 2. API 响应中的时间戳


#### 3. JWT Token 过期时间
JSON Web Token 中的 `exp`（过期时间）和 `iat`（签发时间）字段都是 Unix 时间戳。

#### 4. 日志分析
系统日志中的时间戳需要转换为可读格式才能分析问题发生的时间线。

#### 5. 前后端时间同步调试
前端 JavaScript 使用毫秒级时间戳（`Date.now()`），后端可能使用秒级时间戳（`PHP time()`），需要互转比对。


## 时间戳常见问题（FAQ）

### Q1: 为什么相同的时间在不同时区显示时间戳一样？

因为 Unix 时间戳是相对于 UTC 时间的，全球统一。中国上海和美国纽约在同一物理时刻的时间戳完全相同，只是本地时间显示不同（如 18:00 vs 06:00）。

### Q2: 2038 年问题是什么？

32 位系统中的时间戳使用有符号 32 位整数存储，最大值是 `2147483647`（对应 2038-01-19 03:14:07 UTC）。超过此值会导致整数溢出。64 位系统不受此限制。

### Q3: 时间戳可以表示 1970 年之前的时间吗？

可以。负值时间戳表示 1970 年之前的日期。例如 `-31536000` 对应 1969 年 1 月 1 日。

### Q4: 秒级和毫秒级时间戳如何区分？

- 秒级：10 位数字（如 `1715712000`），范围约 1970-2286 年
- 毫秒级：13 位数字（如 `1715712000000`），范围约 1970-5138 年
- 如果看到 16-19 位数字，通常是微秒或纳秒级

### Q5: 为什么 JavaScript 使用毫秒级而 PHP 使用秒级？

JavaScript 的 `Date.now()` 返回毫秒精度是为了满足浏览器端动画和 UI 更新的精确需求。PHP 的 `time()` 返回秒级精度，因为服务端场景秒级精度通常已足够。
