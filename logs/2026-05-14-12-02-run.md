## 经验教训

1. **Base64 的中文处理**：`btoa()` 不支持直接处理中文（Unicode 字符超出 Latin-1 范围）。必须先用 `encodeURIComponent(text)` 将中文转为 UTF-8 百分号编码，再用 `unescape()` 转为原始字节字符串，最后传给 `btoa()`。解码时反向操作。

2. **JSON 格式化工具样式升级**：之前该工具有交叉链接但无样式化 tools-nav 区域，此次补齐使其与其他工具一致。统一 UI 风格是建立网站品牌的重要步骤。

3. **gh CLI 安装困境**：在 WSL cron 环境下，apt install 因需要 sudo 密码而失败，curl 下载因网络速率极低而失败。GitHub Pages 部署的阻塞条件短期内难以解决。

4. **工具数量已达 4 个**：形成了有意义的工具集，交叉链接网络的 SEO 价值在增加。每次增加工具都更新所有已有工具的链接已成为标准流程。

5. **文件拖拽上传**：实现「点击选择」+「拖拽上传」的双重交互模式，配合视觉反馈（边框变色、状态提示），用户体验更接近原生应用。
