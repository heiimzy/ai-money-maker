## 经验教训

1. **Markdown 预览器适合使用 CDN 库而非自实现解析器**：marked.js 是成熟的 Markdown 解析库，覆盖全部 GFM 语法。自实现解析器不仅开发量大，而且容易遗漏边缘情况（嵌套格式、特殊转义等）。CDN 加载的 marked.js 约 20KB（minified），对页面加载影响极小。

2. **响应式双栏布局需要处理滚动同步**：桌面端双栏左右布局，移动端需要切换为上下布局。768px 断点是合理的阈值，因为宽度不足时双栏会让编辑区过窄。

3. **防抖（debounce）对 Markdown 解析至关重要**：用户连续输入时，每次按键都触发解析和 DOM 更新会导致明显的卡顿。100ms 防抖延迟在响应性和性能之间取得了很好的平衡。

4. **预置示例模板降低使用门槛**：首次访问自动加载一段完整的 Markdown 示例（包含所有语法类型），让新用户立刻看到效果。这种"即开即用"的设计减少了用户的学习成本。

5. **marked.js CDN 版本需要绑定兼容性**：CDN 版本需要注意 marked.js 不同版本之间 API 可能有差异（比如 `marked.parse()` vs `marked()`）。使用官方 CDN 的最新稳定版本即可。

6. **内容文章的内链策略**：Markdown 教程自然地适合链接到其他工具——UUID 生成器用于创建唯一 ID，Base64 编解码用于嵌入图片，密码生成器用于保护敏感文档。工具间的有机关联比强行放置链接更利于 SEO。
