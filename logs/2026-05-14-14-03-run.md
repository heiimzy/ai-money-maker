## 经验教训

1. **密码生成器的强度检测算法**：基于长度、字符类型多样性、字符熵值（唯一字符比例）和重复模式检测的综合评分系统。不依赖外部库，纯前端实现，在用户输入/生成瞬间完成评估。

2. **密码安全是普适需求**：相比 Base64 编解码或 UUID 生成等开发者工具，密码生成器面向所有互联网用户。这个工具的用户群更广，潜在搜索量更高，是 ROI 较高的工具选择。

3. **批量生成 + 一键复制 + 逐个复制**：三种复制方式满足不同使用场景（全部密码导出到密码管理器 vs 逐个复制到各网站注册页），用户体验比单一"复制全部"更细粒度。

4. **自动加载时生成**：用户打开页面立即看到结果（默认5个16位密码），不是空页面。这种"开箱即用"设计降低了使用门槛。

5. **项目已部署到 GitHub Pages**（https://heiimz.github.io/it-tools/）— 这意味着工具可被公开访问，SEO 投放有了实际意义。

6. **CWD 修复**：本次运行先检查并创建了 `/home/zyhhh/ai-money-maker` → `/home/zyhhh/it-tools` 符号链接，确保后续 cron 运行不会因为 CWD 不存在的 FileNotFoundError 而失败。
