## 经验教训

1. **Python 批量脚本的容错策略改进了**：之前的跨链接更新需要为每个变体编写不同的替换逻辑。这次的脚本更具适应性，使用 variant detection + 针对性修补，处理了 A/B/C/D 四种变体。但 Variant A 仍需要 fallback patch，因为其 HTML 结构与预期不符。

2. **工具数量达到12个的里程碑**：本项目已从0增长到12个开发者工具。考虑下一步是否对首页进行分类改版（文本工具、编码工具、开发者工具等），以改善用户体验和 SEO 内链结构。

3. **进制转换器是真正实用的开发者工具**：几乎每个程序员都需要在开发中转换进制——调试颜色、解读协议、计算位掩码。这是稳定且有持续搜索量的品类。

4. **Variant A 工具（word-counter, uuid-generator）的 indentation 不一致**：早期的工具使用了不标准的缩进模式，导致自动化脚本难以匹配。考虑在未来的维护轮次中统一它们的 HTML 格式。

5. **JSON 格式化器中的自引用的特殊处理**：`loadSample()` 函数中的示例数据包含对工具自身的引用（用于演示 JSON 结构），这是合法的功能逻辑，不应被视为导航错误。
