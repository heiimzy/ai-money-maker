## 经验教训

1. **11个工具的交叉链接复杂度**：每次构建新工具需要修改 10-30 个 HTML 文件的三区域（tools-nav、info-section、footer）。Python 脚本可以加速 batch 操作，但不同工具 HTML 变体的增加要求脚本具有更强的容错能力。

2. **不同工具 HTML 结构变体增至 4 种**：
   - **早期**（word-counter, uuid-generator）：tools-nav + 简单版权 footer
   - **中期**（json-formatter）：tools-nav + info-section + inline-style footer
   - **中后期**（markdown-previewer, password-generator）：pill-group + info-list + footer-sep（`|` 分隔符）
   - **现代**（其余工具）：tools-nav + info-section + footer（`·` 分隔符）

3. **正则表达式测试器是开发者必备工具**：几乎所有主流工具站都有 regex tester，说明这是稳定流量的品类。功能对比看，我们的工具在匹配高亮、分组展示、替换模式、速查表方面有完整的覆盖。

4. **Python subn vs sub**：使用 `re.subn()` 而不是 `re.sub()` 来同时获取替换结果和替换计数。这是正则脚本开发中容易忽略的细节。

5. **JSON-LD 自引用不是 bug**：JSON-LD 结构化数据中的自链接是 SEO 的预期行为，不应被视为导航错误。audit 脚本需要区分导航链接和 JSON-LD 中的链接。
